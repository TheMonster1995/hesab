02

The feature set

Grouped by the surface each one lives on.

Accounts & Access

Who gets in, and how much they can touch.

  • Email + password login (magic-link optional)
  • Invite by email with a join token
  • Roles: admin vs member per group
  • Read-only share link for a group view

Groups & Members

The container the money moves inside.

  • Create & name groups
  • Add members (invited users or plain names)
  • A person belongs to many groups
  • Global balance across all groups

Expenses & Splitting

The core act, and every fair way to divide it.

  • Amount, description, payer, date, notes
  • Equal & exact-amount splits
  • Percentage & shares/weights
  • Adjustment (+/- on equal)
  • Itemized split heavier — late phase
  • Multi-currency per expense
  • Edit & delete with change history

Balances & Settling

The reckoning — and the way out of it.

  • Live "who owes whom" per group
  • Debt simplification to fewest transfers
  • Record a settle-up payment
  • Reimbursement tracking (settled off-app)

Data & Experience

Living with the app day to day.

  • Search & filter the expense history
  • Export to CSV / PDF
  • Dark mode
  • Mobile-responsive, installable PWA

Operations

Keeping the self-hosted instance healthy.

  • Email notifications added / paid
  • Data backup & restore
  • Single Docker Compose deploy
  • Postgres as the source of truth
03

The stack

A Vite SPA talking GraphQL to an Express API.

A monorepo with two deployables — a Vite + React client and an Express + GraphQL API — sharing one Postgres database. The GraphQL schema is the contract between them. Prisma handles data access; auth is stateless JWT, with argon2 for password hashing.

FrontendVite · React + TS
StylingTailwind CSS
APIExpress + GraphQL · GraphQL Yoga
DatabasePostgreSQL
ORMPrisma
AuthJWT · argon2
DeployDocker Compose · web + api + db

05

Execution plan

Each phase ends on something you can actually use.

Vertical slices, not layers. The app does something real by Phase 3 and everything after is depth. Ship, use it with the group, then keep going.

0Setup

Scaffold & self-host skeleton

Monorepo: a Vite + React client and an Express + GraphQL API, Prisma wired to Postgres, Docker Compose bringing up web + api + db, base Tailwind layout with dark-mode tokens.

Accounts & login

JWT auth on the GraphQL API — email + password hashed with argon2, signup / login / logout mutations that issue a signed token, verified in the GraphQL context with resolver-level guards on protected fields.
